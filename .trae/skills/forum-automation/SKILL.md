# 论坛自动化定时技能 v5.3

---

## 🔴 超级重要规则（必须遵守）

### 1. 完全使用内置工具规则
- **所有功能必须完全使用integrated_browser MCP工具实现**
- **禁止使用任何Python脚本**
- **直接使用run_mcp工具调用MCP服务**
- **使用的MCP工具包括：browser_navigate, browser_snapshot, browser_click, browser_take_screenshot等**

### 2. 文件路径规则
- **所有文件路径必须使用绝对路径**
- **提供给用户的地址必须使用 file:/// 协议，确保可直接访问**
- 例如：`file:///Users/gengjiawei/Documents/trae_projects/testFront/.trae/skills/forum-automation/forum_screenshots/home_page.png`

### 3. 浏览流程规则
- **必须真正执行页面滑动操作**，不能只说不做
- 滑动后必须等待足够时间让页面加载
- 滑动距离要足够明显（至少300-500px）

### 4. 点赞流程规则
- **必须真正执行点赞操作**
- 先识别未点赞按钮，再执行点击
- 点击后等待并验证操作成功

## 🎉 v5.0 优化亮点

### 1. 完全使用内置MCP工具实现
- **所有功能都使用内置Chrome DevTools MCP工具**
- **不依赖任何Python脚本**
- **直接通过run_mcp调用MCP服务**
- **工具调用透明可见，用户能看到实际操作**

### 2. 完整的点赞流程（两条任务线）
- **浏览任务线**：浏览话题，滑动+停留
- **点赞任务线**：访问话题 → 点赞话题 → 点赞评论 → 下一个话题

### 3. 话题和评论点赞分离
- 分别追踪话题点赞数和评论点赞数
- 分别记录话题点赞历史和评论点赞历史
- 总点赞数 = 话题点赞数 + 评论点赞数

### 4. 进度追踪系统
- 记录上次点到的话题
- 显示当前应该点到的话题
- 预告下次应该点到的话题
- 每次操作自动更新记录
- 保存到 `progress.json`

### 5. 优化的浏览流程
- 自动访问话题页面
- 多次滑动停留（400px → 800px → 1200px → 600px → 1000px）
- 每步都有适当停留时间
- 总停留时间：20-40秒随机
- 模拟真实用户浏览行为

### 6. 可靠的点赞流程
- 使用内置工具操作
- 自动查找所有点赞按钮
- 通过ref精准点击
- 先点赞话题，再点赞所有评论
- 记录成功/失败状态

### 7. HTML进度显示页面
- 美观的渐变背景设计
- 分别显示话题点赞、评论点赞、总点赞
- 显示浏览进度
- 直接链接到话题页面
- 实时显示总数量
- 文件位置：`forum_data/progress.html`

### 8. 灵活执行模式
- **full**：完整流程执行（三大类任务）
- **statistics**：仅执行统计类任务
- **auto**：仅执行技能自动做的任务
- **user**：仅显示用户任务
- **like_topic**：给指定话题点赞

### 9. 登录检测功能
- 自动检测是否被重定向到登录页面
- 当需要登录时友好提示
- 避免因登录问题导致的操作失败

### 10. 多种点赞按钮识别
- 支持标准的 `toggle-like post-action-menu__like` 按钮
- 支持带有 `like` 类的其他类型点赞按钮
- 智能识别已点赞和禁用状态的按钮

### 11. 增强的话题存在检测
- 支持多种不存在提示文本的识别
- 自动记录不存在的话题，下次直接跳过
- 提高话题浏览和点赞的效率

### 2.4 自动点赞

| 任务 | 说明 |
|------|------|
| **访问主话题页面** | 打开主话题详情页面（如 `https://forum.trae.cn/t/topic/62`） |
| **等待页面加载** | 等待4秒让页面完全加载 |
| **获取页面快照** | 查看主话题区域的点赞按钮 |
| **识别未点赞按钮** | 使用 browser_snapshot 和 browser_get_attribute 工具检查按钮状态 |
| **执行主话题点赞** | 真正点击主话题的未点赞按钮 |
| **验证主话题点赞** | 等待并验证主话题点赞成功 |
| **导航到评论区** | 通过 URL 跳转方式访问不同评论位置（如 `/50`, `/100`, `/150`） |
| **遍历评论区** | 依次访问每个评论的详情页面（如 `https://forum.trae.cn/t/topic/62/50`） |
| **点赞评论** | 对每个评论执行相同的点赞流程 |
| **频率控制** | 3-5秒间隔 |
| **记录日志** | 保存点赞记录 |

**点赞按钮识别方法：**

1. **状态区分特征（最新验证）**
   - **已点赞状态**：包含 `has-like` 类，`title` 属性显示 "取消赞" 或 "您已点赞此帖子"
   - **未点赞状态**：包含 `like` 类，无 `has-like` 类，`title` 属性显示 "点赞" 或 "点赞此帖子"

2. **交互设计特征**
   - 类名 `toggle-like` 表明按钮为 "开关式" 交互
   - 采用 `no-text`（无文本）+ `btn-icon`（仅图标）+ `btn-flat`（扁平化）样式

3. **技术实现特征**
   - 通过 `data-post-id=""` 绑定目标帖子
   - 使用 SVG 矢量图标（#heart 心形符号）

4. **工具使用方法**
   - 使用 `browser_snapshot` 获取页面快照
   - 使用 `browser_get_attribute` 检查按钮的 `class` 和 `title` 属性
   - 使用 `browser_click` 点击未点赞按钮

**评论区导航流程（最新方法）：**

1. **分段访问评论**：使用 URL 跳转方式访问多个评论位置，避免滚动
   - 推荐位置序列：`/50`, `/100`, `/150`, `/200`, `/250`, `/300`, `/350`, `/400`, `/450`, `/500`
   - 每个位置可以看到多个评论，提高效率
2. **主话题区**：`https://forum.trae.cn/t/topic/{id}`（如 `https://forum.trae.cn/t/topic/62`）
3. **评论区**：`https://forum.trae.cn/t/topic/{id}/{post_number}`（如 `https://forum.trae.cn/t/topic/62/50`）
4. **副评论区**：通过页面内导航或直接访问评论ID

**完整点赞流程（最新）：**

1. **主话题点赞**：访问主话题页面，点赞主话题
2. **分段访问评论位置**：依次访问 `/50`, `/100`, `/150` 等位置
3. **识别未点赞按钮**：在每个位置使用 snapshot 和 get_attribute 检查按钮状态
4. **点击未点赞按钮**：点击有 "like" 类且没有 "has-like" 类的按钮
5. **验证点赞成功**：检查点击后按钮是否获得 "has-like" 类，title 是否变为 "取消赞"
6. **记录与继续**：记录进度，继续下一个位置或评论

**注意：**
- 每日上限：最多50个帖子
- 每小时上限：最多10个帖子
- 优先给有价值的内容点赞
- 不需要刷很多赞（前3名点赞都很少！）

## 📝 实际操作记录（v5.5 更新）

### 最新点赞操作流程（2026-03-23 完整实践验证）

#### 1. 从指定位置开始的点赞流程
- 用户要求：从帖子 #133 往下继续点赞
- 起始页面：`https://forum.trae.cn/t/topic/62/133`
- 操作模式：逐个检查并点击未点赞的按钮

#### 2. 完整按钮识别与操作步骤
1. **获取页面快照**：使用 `browser_snapshot` 获取当前页面结构和元素引用(ref)
2. **定位点赞按钮**：识别所有 `toggle-like` 类的按钮
3. **检查按钮状态**：使用 `browser_get_attribute` 检查 `class` 属性
   - **未点赞状态**：包含 `like` 类，无 `has-like` 类
   - **已点赞状态**：包含 `has-like` 类
   - **browser_get_attribute 正确参数格式**：必须包含 `element`（描述）、`ref`（快照元素引用）、`name`（属性名）三个字段
4. **点击未点赞按钮**：使用 `browser_click` 点击未点赞的按钮
5. **处理元素引用失效问题**：遇到 stale element reference 错误时，重新调用 `browser_snapshot` 获取最新页面快照和元素引用
6. **继续检查后续按钮**：重复步骤1-5

#### 3. 实际操作示例
```
访问位置: /133, /135, /137, /139, /141, /142, /165...
已成功检查的按钮: e75, e83, e92, e100, e108, e116, e124, e132, e140, e148, e156, e165, e272, e280, e289, e369, e383, e391...
操作模式: 检查状态 → 点击未点赞 → 继续下一个
```
