# 页面视觉识别技能 (Page Visual Inspector)

让AI能够"看到"和分析网页界面的技能！

## 功能说明

此技能利用浏览器MCP工具，让AI可以：

- 打开浏览器访问特定页面（本地或远程）
- 截取页面完整截图或特定元素截图
- 获取页面HTML结构和DOM快照
- 分析页面的视觉样式和布局
- 识别界面组件、配色方案、排版等

## 核心能力

### 1. 页面导航与访问
- 支持本地开发服务器页面（localhost）
- 支持远程URL访问
- 自动等待页面加载完成

### 2. 截图功能
- 完整页面截图
- 视口截图
- 特定元素截图（通过CSS选择器）

### 3. DOM分析
- 获取页面可访问性快照（比截图更适合AI分析）
- 读取页面HTML内容
- 识别页面结构和组件

### 4. 样式识别
- 分析页面配色方案
- 识别UI组件类型
- 理解布局结构

## 使用场景

### 何时调用此技能

- 用户说："看看这个页面的样式"
- 用户说："分析一下这个界面"
- 用户说："这个页面长什么样？"
- 用户上传了设计稿，要求AI参照实现
- 需要AI根据现有页面进行样式调整或重构
- 调试UI问题时需要AI查看当前状态

## 使用步骤

### 标准工作流程

1. **确认目标页面**
   - 确定要查看的URL（本地或远程）
   - 如果是本地项目，先检查开发服务器是否运行

2. **浏览器导航**
   - 使用 browser_navigate 或 playwright_navigate 访问页面
   - 等待页面完全加载

3. **获取页面信息**
   - 先使用 browser_snapshot 获取可访问性快照（AI更容易理解）
   - 可选：使用 browser_take_screenshot 获取截图
   - 可选：使用 playwright_get_visible_html 获取HTML结构

4. **分析与反馈**
   - 基于获取的信息分析页面样式
   - 识别组件、配色、布局等
   - 提供详细的视觉描述

## 可用MCP工具

### integrated_browser 工具
- `browser_tabs` - 查看当前打开的标签页
- `browser_navigate` - 导航到URL
- `browser_snapshot` - 获取页面可访问性快照（推荐用于AI分析）
- `browser_take_screenshot` - 截取页面截图
- `browser_wait_for` - 等待页面元素加载

### mcp_Playwright 工具
- `playwright_navigate` - 导航到URL
- `playwright_screenshot` - 截图（支持base64返回）
- `playwright_get_visible_html` - 获取页面HTML
- `playwright_get_visible_text` - 获取页面文本

## 实际使用示例

### 示例1：查看本地开发页面
```
用户："看看我本地的登录页面长什么样"
执行步骤：
1. 检查本地开发服务器是否运行
2. 浏览器导航到 http://localhost:3000/login
3. 获取页面快照和截图
4. 分析并描述页面样式
```

### 示例2：分析设计稿并实现
```
用户："参照这个设计稿实现页面"
执行步骤：
1. 访问设计稿URL或打开本地文件
2. 截取设计稿截图
3. 分析设计稿的视觉元素
4. 根据分析结果实现代码
```

### 示例3：调试UI问题
```
用户："帮我看看按钮为什么显示不对"
执行步骤：
1. 打开问题页面
2. 获取当前页面状态
3. 分析按钮元素的样式
4. 找出问题并提供修复方案
```

## 最佳实践

1. **优先使用快照**：browser_snapshot 比截图更适合AI理解页面结构
2. **组合使用**：快照 + 截图 + HTML，提供全面的页面信息
3. **等待加载**：确保页面完全加载后再获取信息
4. **详细描述**：分析时要具体说明颜色、布局、组件类型等

## 注意事项

- 如果是本地项目，确保开发服务器正在运行
- 截图功能不能直接用于AI视觉识别，主要用于用户查看
- 可访问性快照（browser_snapshot）是AI分析页面的最佳选择
- 可以组合使用多个工具获取更全面的页面信息
