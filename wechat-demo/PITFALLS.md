# 微信小程序开发常见坑点

## 1. 登录鉴权

### 1.1 微信登录流程
- **坑点**: `wx.login` 获取的 code 只能使用一次，且有时效性
- **坑点**: 获取手机号必须使用 `button` 组件的 `open-type="getPhoneNumber"`，无法通过 API 直接调用
- **坑点**: 个人小程序无法使用获取手机号功能

### 1.2 登录状态维护
- **坑点**: token 需要本地存储，但小程序没有 cookie 机制
- **坑点**: 需要处理 token 过期后的重新登录逻辑

## 2. 支付功能

### 2.1 微信支付限制
- **坑点**: 个人小程序无法开通支付功能
- **坑点**: 支付参数需要从后端获取，前端不能生成
- **坑点**: 支付回调通知需要后端处理，前端无法可靠接收

### 2.2 支付流程
- **坑点**: `wx.requestPayment` 在开发者工具中无法真实支付
- **坑点**: 支付成功后需要主动查询订单状态，不能依赖回调

## 3. WebView 限制

### 3.1 使用限制
- **坑点**: 个人类型小程序无法使用 WebView
- **坑点**: WebView 页面无法调用小程序 API
- **坑点**: WebView 和小程序之间通信只能通过 URL 参数和 JSSDK

### 3.2 域名限制
- **坑点**: WebView 只能打开业务域名下的页面
- **坑点**: 业务域名需要在小程序后台配置，且需要 ICP 备案

## 4. 网络请求

### 4.1 域名限制
- **坑点**: 只能请求已配置的域名，不支持 IP 地址
- **坑点**: 开发版可以开启不校验域名，但体验版和正式版必须配置
- **坑点**: HTTPS 证书必须有效且符合 ATS 标准

### 4.2 并发限制
- **坑点**: `wx.request` 最大并发 10 个
- **坑点**: WebSocket 最大并发 5 个
- **坑点**: 小程序进入后台后，5秒内未完成的请求会被中断

## 5. 权限管理

### 5.1 授权机制
- **坑点**: 首次拒绝授权后，再次调用 API 会直接失败，需要引导用户去设置页
- **坑点**: `wx.authorize` 只能触发一次授权弹窗
- **坑点**: 部分权限（如获取手机号）无法通过 `wx.authorize` 申请

### 5.2 权限列表
- 用户信息、位置、相册、相机、麦克风等都需要单独授权
- iOS 和 Android 的权限提示时机和方式不同

## 6. 组件限制

### 6.1 原生组件
- **坑点**: map、video、canvas、textarea 等是原生组件，层级最高
- **坑点**: 原生组件上无法覆盖其他元素（除 cover-view/cover-image）
- **坑点**: 原生组件在 scroll-view 中可能表现异常

### 6.2 cover-view
- **坑点**: cover-view 只能嵌套在 map、video、canvas、camera 等组件中
- **坑点**: cover-view 内部只能使用 cover-view 和 cover-image
- **坑点**: cover-view 不支持 overflow: hidden

## 7. 样式限制

### 7.1 CSS 限制
- **坑点**: 不支持部分 CSS3 特性（如 flex: 1 在某些情况下不生效）
- **坑点**: 不支持 * 选择器
- **坑点**: 不支持属性选择器（如 [data-type="xxx"]）
- **坑点**: 不支持 CSS 变量（CSS 自定义属性）

### 7.2 rpx 单位
- **坑点**: rpx 在不同设备上可能产生 1px 偏差
- **坑点**: 字体大小小于 24rpx 在某些设备上可能不生效

## 8. 存储限制

### 8.1 本地存储
- **坑点**: 本地存储总容量限制为 10MB
- **坑点**: 单个 key 最大 1MB
- **坑点**: 同步存储会阻塞主线程，大量数据建议使用异步 API

## 9. 文件系统

### 9.1 文件操作
- **坑点**: 小程序有独立的文件系统，与系统文件隔离
- **坑点**: 临时文件需要手动清理，否则可能超出存储限制
- **坑点**: 文件路径格式与 Web 不同（如 wxfile://）

## 10. 其他限制

### 10.1 包大小限制
- **坑点**: 整个小程序所有分包总大小不超过 20MB
- **坑点**: 单个分包/主包大小不超过 2MB
- **坑点**: 图片、字体等资源都会计入包大小

### 10.2 页面栈限制
- **坑点**: 页面栈最多 10 层，超过会无法跳转
- **坑点**: 需要合理使用 `redirectTo` 和 `navigateBack`

### 10.3 跳转限制
- **坑点**: `navigateTo` 最多 10 层，超过需要 `redirectTo`
- **坑点**: 不能跳转到 tabBar 页面，需要使用 `switchTab`

### 10.4 分享限制
- **坑点**: 页面默认没有分享功能，需要手动开启
- **坑点**: 分享图片有尺寸限制（5:4，宽度 500px）

## 11. 开发工具与真机差异

### 11.1 常见差异
- **坑点**: 开发者工具不校验域名，真机会校验
- **坑点**: 开发者工具不支持获取手机号
- **坑点**: 开发者工具不支持扫码
- **坑点**: 某些 API 在开发工具和真机上表现不同

### 11.2 调试建议
- 务必在真机上测试
- 使用真机调试功能
- 注意查看控制台报错

## 12. 审核相关

### 12.1 审核限制
- **坑点**: 内容需要符合微信小程序运营规范
- **坑点**: 某些功能需要特定类目才能使用
- **坑点**: 个人和企业小程序的功能权限不同

### 12.2 常见被拒原因
- 涉及支付但没有相应类目
- 涉及用户生成内容但没有审核机制
- 功能不完整或存在明显 bug
- 页面内容为空或加载失败

## 13. 性能优化

### 13.1 列表渲染
- **坑点**: 长列表需要虚拟滚动，否则性能很差
- **坑点**: `wx:for` 需要设置 `wx:key`，否则可能渲染异常

### 13.2 图片优化
- **坑点**: 大图片会导致内存占用过高
- **坑点**: 建议使用 CDN 和压缩后的图片
- **坑点**: 图片懒加载需要手动实现

## 14. 兼容性

### 14.1 基础库版本
- **坑点**: 不同基础库版本 API 支持不同
- **坑点**: 需要使用 `wx.canIUse` 判断 API 兼容性
- **坑点**: 低版本基础库可能不支持某些新特性

### 14.2 平台差异
- **坑点**: iOS 和 Android 表现可能有差异
- **坑点**: 字体渲染、动画性能等平台差异

## 15. 最佳实践建议

1. **开发阶段**
   - 使用 ESLint 规范代码
   - 合理使用组件化开发
   - 做好错误处理和边界情况处理

2. **测试阶段**
   - 真机测试必不可少
   - 测试各种网络环境
   - 测试不同基础库版本

3. **上线阶段**
   - 配置好服务器域名
   - 配置好业务域名（WebView）
   - 准备好审核所需的材料

4. **维护阶段**
   - 监控线上错误
   - 及时更新基础库版本
   - 关注微信官方公告
