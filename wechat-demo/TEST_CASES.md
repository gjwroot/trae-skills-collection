# 微信小程序测试用例

## 1. 请求封装测试 (utils/request.js)

### 1.1 正常请求测试
- **测试场景**: 发送正常的 GET 请求
- **预期结果**: 
  - 显示 loading
  - 请求成功后隐藏 loading
  - 返回正确的数据
  - resolve 被调用

### 1.2 POST 请求测试
- **测试场景**: 发送 POST 请求
- **预期结果**:
  - 正确发送 POST 请求
  - 请求体正确传递
  - 返回正确的数据

### 1.3 Token 注入测试
- **测试场景**: 已登录用户发送请求
- **预期结果**:
  - 请求头中包含 Authorization: Bearer {token}
  - Token 从 storage 正确读取

### 1.4 HTTP 401 测试
- **测试场景**: 服务器返回 HTTP 401
- **预期结果**:
  - 清除 token
  - 清除 userInfo
  - 跳转到登录页 (wx.reLaunch)
  - 显示错误提示

### 1.5 业务 401 测试
- **测试场景**: 业务状态码返回 401
- **预期结果**:
  - 清除 token
  - 清除 userInfo
  - 跳转到登录页 (wx.reLaunch)
  - 不显示错误提示

### 1.6 网络错误测试
- **测试场景**: 网络连接失败
- **预期结果**:
  - 隐藏 loading
  - 显示 "Network connection failed" 提示
  - reject 被调用

### 1.7 Falsy 值测试
- **测试场景**: 返回 data.data 为 0, false, null, ""
- **预期结果**:
  - 正确返回 falsy 值，不替换为 {}
  - resolve(data.data !== undefined ? data.data : {})

### 1.8 Loading 控制测试
- **测试场景**: 设置 showLoading: false
- **预期结果**:
  - 不显示 loading
  - 不调用 wx.showLoading
